# Day 3 Lab Guide: GitOps Secrets Management with Sealed Secrets

In this lab, you will configure **Bitnami Sealed Secrets** to safely encrypt sensitive environment variables (such as API keys and database credentials) so they can be pushed to public or private Git repositories without risk of exposure.

---

## 🔒 The Concept of Sealed Secrets
Standard Kubernetes Secrets are base64-encoded, which is easily decoded. **Sealed Secrets** solves this:
1. You write a standard `Secret` locally.
2. You run `kubeseal`, which encrypts the secret using the cluster's public key.
3. The resulting `SealedSecret` is safe to commit to Git.
4. The `SealedSecret` controller running inside your cluster decrypts the values into a standard Kubernetes `Secret`.

---

## 🛠️ Step 1: Install the Sealed Secrets Controller
Install the controller in your cluster (e.g. k3s/Minikube) using Helm:
```bash
helm repo add sealed-secrets https://bitnami-labs.github.io/sealed-secrets
helm install sealed-secrets-controller sealed-secrets/sealed-secrets -n kube-system
```

Install the local client CLI (`kubeseal`):
```bash
# macOS
brew install kubeseal

# Linux
KUBESEAL_VERSION="0.26.0" # Use latest version
curl -L "https://github.com/bitnami-labs/sealed-secrets/releases/download/v${KUBESEAL_VERSION}/kubeseal-${KUBESEAL_VERSION}-linux-amd64.tar.gz" | tar -xzf - kubeseal
sudo install -m 755 kubeseal /usr/local/bin/kubeseal
```

---

## 🛠️ Step 2: Create and Seal a Secret

1. Generate a raw secret locally (do NOT commit this file to Git):
   ```bash
   kubectl create secret generic tenant1-secrets \
     --from-literal=TAVILY_API_KEY="tvly-your-actual-key" \
     --from-literal=DATABASE_URL="postgresql://user:pass@host:5432/db" \
     --dry-run=client -o yaml > temp-secret.yaml
   ```

2. Seal the secret using the cluster key:
   ```bash
   kubeseal --format yaml < temp-secret.yaml > charts/tenant1/templates/sealedsecret.yaml
   ```

3. Clean up the plaintext secret:
   ```bash
   rm temp-secret.yaml
   ```

---

## 🛠️ Step 3: Reference the Decrypted Secret in Helm Deployment

Update `charts/tenant-app/templates/deployment.yaml` to load environment variables from the generated secret:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ .Release.Name }}-deployment
spec:
  template:
    spec:
      containers:
        - name: app-container
          image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}"
          env:
            # Map Tavily Key from the Secret generated by the SealedSecret controller
            - name: TAVILY_API_KEY
              valueFrom:
                secretKeyRef:
                  name: {{ .Release.Name }}-secrets
                  key: TAVILY_API_KEY
            # Map Database URL
            - name: DATABASE_URL
              valueFrom:
                secretKeyRef:
                  name: {{ .Release.Name }}-secrets
                  key: DATABASE_URL
```

---

## 🧪 Verification & Testing
1. Push `sealedsecret.yaml` to your Git repository.
2. Observe Argo CD sync the resource.
3. Verify the standard Secret is decrypted in the namespace:
   ```bash
   kubectl get secrets -n tenant1
   ```
4. Verify the pods successfully load the environment credentials and connect to external APIs.
